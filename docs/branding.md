# Funature Naming Guide

Funature is the public product name.

## Public Names

| Item | Use |
|------|-----|
| Product | Funature |
| Current public domain | `https://www.funature.fun` |
| Main product area | Funature Work |
| Future areas | Funature Home, Funature Muse |

## Legacy Internal Names

Some names remain for compatibility and should only appear when they refer to real
implementation details:

| Legacy name | Keep when referring to |
|-------------|------------------------|
| `ai-filter` | Existing PM2 process names, nginx config, old server paths, historical changelog entries |
| `imagebase` | Existing repository slug, local filesystem root `~/.imagebase`, old environment variables |
| `window.ImageBase` | Current runtime namespace for the Funature Demo SDK |
| `AI Filter` | Historical early-stage feature/project references only |

## Writing Rule

Use **Funature** in README, product copy, design docs, launch docs, screenshots,
and future-facing architecture docs. Use a legacy name only when changing it would
misrepresent the current code or deployment contract.
